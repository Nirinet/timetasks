# מסמך איפיון מערכת TimeTask — גרסה 1.0

> **תאריך עדכון אחרון:** מרץ 2026
> **כתובת:** https://timetask.sitechs.co.il

---

## 1. מטרת המערכת

**TimeTask** היא מערכת ניהול משימות ומעקב זמן מלאה בעברית, המיועדת לחברות פיתוח תוכנה.

המערכת מאפשרת:
- **ניהול פרויקטים** — יצירה, מעקב וניהול פרויקטים עבור לקוחות שונים
- **ניהול משימות** — הקצאת משימות לעובדים וללקוחות, מעקב סטטוס ועדיפויות, תמיכה במשימות-בת (היררכיה)
- **מעקב זמן** — טיימר חי (Start/Stop), הזנת שעות ידנית, דוחות שעות מפורטים
- **שיתוף פעולה** — תגובות על משימות ופרויקטים, העלאת קבצים, התראות בזמן אמת
- **דוחות** — דוחות שעות, סטטוס פרויקטים, ביצועי עובדים, משימות פתוחות עם ייצוא ל-PDF/Excel/CSV
- **תצוגות מתקדמות** — טבלה, קנבן (Drag & Drop), לוח שנה, גאנט, תצוגת צוות

**קהל יעד:** מנהלי חברת פיתוח, עובדים/מפתחים, ולקוחות שרוצים לעקוב אחרי הפרויקטים שלהם.

## 2. ארכיטקטורה טכנית

### Stack טכנולוגי

| שכבה | טכנולוגיה |
|------|-----------|
| צד לקוח (Frontend) | React 18 + TypeScript, Vite, MUI v5 (Material UI) |
| צד שרת (Backend) | Node.js + Express + TypeScript |
| מסד נתונים | PostgreSQL + Prisma ORM |
| זמן אמת | Socket.IO |
| אימות | JWT (Access Token + Refresh Token) |
| שליחת מיילים | Nodemailer (SMTP) |
| משימות מתוזמנות | node-cron |
| פריסה | Docker Compose + Cloudflare Tunnel |
| ניטור | Prometheus metrics endpoint |

### תמיכה בעברית (RTL)
- כיוון RTL מובנה (direction: rtl) בכל הרכיבים
- פונט Heebo (Google Fonts)
- תאריכים בפורמט עברי (he-IL)
- תמיכה בעברית בייצוא PDF, Excel ו-CSV (כולל BOM)

## 3. דרגות משתמשים (תפקידים)

המערכת תומכת ב-3 דרגות:

| תפקיד | תיאור |
|--------|--------|
| **ADMIN** (מנהל מערכת) | שליטה מלאה: ניהול משתמשים, לקוחות, פרויקטים, משימות, הגדרות מערכת, דוחות מלאים |
| **EMPLOYEE** (עובד) | ניהול לקוחות, פרויקטים ומשימות. מעקב זמן. דוחות. אין ניהול משתמשים או הגדרות מערכת |
| **CLIENT** (לקוח) | צפייה בפרויקטים ומשימות המוקצים לו בלבד. תגובות והעלאת קבצים. צפייה בדוחות מוגבלים. אין גישה לניהול לקוחות, משתמשים או הגדרות |

## 4. יכולות לפי דרגת משתמש

### 4.1 מנהל מערכת (ADMIN)

**תפריט צד (Sidebar):**
דשבורד, משימות, פרויקטים, לקוחות, מעקב זמן, דוחות, משתמשים, הגדרות מערכת, פרופיל

**דשבורד:**
- 6 כרטיסי סיכום: סך משתמשים, סך פרויקטים, סך משימות, משימות פעילות, משימות באיחור, שעות החודש
- 5 משימות אחרונות (טבלה)
- 5 פרויקטים אחרונים (טבלה)
- רשימת עובדים פעילים

**ניהול משתמשים:**
- צפייה ברשימת כל המשתמשים (שם, מייל, תפקיד, סטטוס, טלפון, תאריך הצטרפות)
- יצירת משתמש חדש (כולל הגדרת סיסמה ותפקיד)
- עריכת משתמש (שם, טלפון, תפקיד, סטטוס פעיל/לא פעיל)
- השבתת משתמש (Soft Delete — מבטל גישה מבלי למחוק נתונים)
- לא ניתן להשבית את עצמו

**ניהול לקוחות:**
- צפייה, יצירה, עריכה ומחיקה של לקוחות
- מחיקת לקוח אפשרית רק אם אין לו פרויקטים מקושרים
- צפייה בפרטי לקוח כולל פרויקטים מקושרים

**ניהול פרויקטים:**
- צפייה, יצירה, עריכה ומחיקה של פרויקטים
- יכול למחוק פרויקט גם אם יש בו משימות (מחיקה מדורגת מלאה)
- יצירת פרויקט מתבנית
- הוספת תגובות על פרויקט
- סינון לפי סטטוס (פעיל / מושהה / הושלם)

**ניהול משימות:**
- כל 5 תצוגות: טבלה, קנבן, לוח שנה, גאנט, צוות
- יצירה, עריכה, מחיקה ושכפול משימות
- הקצאת משימות לעובדים וללקוחות
- סינון לפי פרויקט, סטטוס ועדיפות
- 4 טאבים בפרטי משימה: פרטים, תגובות, קבצים, רשומות זמן
- יצירת משימות-בת (Subtasks)

**מעקב זמן:**
- הפעלה/עצירה של טיימר (עד 3 טיימרים פעילים)
- הזנת שעות ידנית
- צפייה בכל רשומות הזמן (כולל סינון לפי עובד)
- מחיקת רשומות זמן

**דוחות:**
- דוח שעות (עם סינון תאריכים ועובד)
- סטטוס פרויקטים
- ביצועי עובדים (שעות, משימות, אחוזי השלמה)
- משימות פתוחות (כולל חריגות deadline)
- ייצוא לכל הפורמטים: PDF, Excel, CSV

**הגדרות מערכת:**
- שם המערכת
- סף התראת טיימר שכוח (דקות)
- חלון התראת deadline מתקרב (שעות)

**חיפוש גלובלי:**
- חיפוש חופשי במשימות, פרויקטים ולקוחות מסרגל החיפוש העליון

**התראות:**
- קבלת כל סוגי ההתראות (9 סוגים)
- סימון כנקראו (בודד או כולם)

דשבורד, משימות, פרויקטים, לקוחות, מעקב זמן, דוחות, פרופיל

**דשבורד:**

**ניהול לקוחות:**

**ניהול משימות:**

- יצירה, עריכה ושכפול

**מעקב זמן:**

**דוחות:**

**חיפוש גלובלי:**

### 4.3 לקוח (CLIENT)

**תפריט צד:**
דשבורד, משימות, פרויקטים, מעקב זמן, דוחות, פרופיל

**דשבורד:**
- 4 כרטיסי סיכום: הפרויקטים שלי, המשימות שלי, משימות הושלמו, שעות החודש
- הפרויקטים שלי (עם אחוזי השלמה)
- **"ממתינות לתגובתך"** — משימות בסטטוס WAITING_CLIENT (בולט בצהוב)
- פעילות אחרונה
- פעולות מהירות (מידע)

**פרויקטים:**
- **צפייה בלבד** בפרויקטים שבהם יש לו משימות מוקצות
- לא יכול ליצור, לערוך או למחוק פרויקטים

**משימות:**
- צפייה במשימות המוקצות לו בלבד (כל 5 התצוגות)
- יכול ליצור משימה (רק בפרויקטים שיש לו גישה)
- יכול לעדכן סטטוס משימה
- יכול לשכפל משימה
- **לא יכול למחוק משימות**
- יכול להוסיף תגובות ולהעלות קבצים

**מעקב זמן:**
- **צפייה בלבד** ברשומות זמן של משימות המוקצות לו
- לא יכול להפעיל טיימר, להזין שעות ידנית, או למחוק רשומות

**דוחות:**
- דוח שעות (מסונן לפרויקטים שלו)
- סטטוס פרויקטים (מסונן לפרויקטים שלו)
- משימות פתוחות (מסונן למשימות שלו)
- **אין גישה לדוח ביצועי עובדים**
- ייצוא: PDF, Excel, CSV

**חיפוש גלובלי:**
- חיפוש במשימות ופרויקטים שלו בלבד
- לא מחפש בלקוחות

**אין גישה ל:** ניהול לקוחות, ניהול משתמשים, הגדרות מערכת, הפעלת טיימרים

## 5. מודולי המערכת — איפיון מפורט

### 5.1 אימות והרשאות

**התחברות:**
- כניסה עם מייל וסיסמה
- JWT — שני טוקנים: Access Token (7 ימים) ו-Refresh Token (30 ימים)
- המשתמש הראשון שנרשם מקבל אוטומטית תפקיד ADMIN
- הרשמת משתמשים נוספים דורשת הרשאת ADMIN

**איפוס סיסמה:**
- לינק "שכחת סיסמה?" בדף ההתחברות
- הזנת כתובת מייל → שליחת מייל עם קישור איפוס (תוקף: שעה אחת)
- הטוקן נשמר מוצפן (SHA-256) במסד הנתונים
- לאחר שימוש, הטוקן נמחק
- מחזיר תמיד "הצלחה" (גם אם המייל לא קיים) — למניעת חשיפת מידע

**מדיניות סיסמאות:**
- מינימום 8 תווים
- חובה: אות קטנה, אות גדולה, ספרה

**פרופיל:**
- עדכון שם, טלפון
- העדפות: התראות מייל, התראות טיימר
- שינוי סיסמה (דורש סיסמה נוכחית)

- צפייה בפרטי לקוח כולל פרויקטים מקושרים

### 5.3 ניהול פרויקטים

**שדות פרויקט:**
| שדה | חובה | תיאור |
|------|-------|--------|
| שם הפרויקט | כן | |
| תיאור | לא | טקסט חופשי |
| לקוח | כן | בחירה מרשימת הלקוחות |
| תאריך התחלה | אוטומטי | ברירת מחדל: היום |
| תאריך יעד | לא | Deadline כללי לפרויקט |
| תקציב שעות | לא | מספר שעות מתוכננות |
| סטטוס | אוטומטי | פעיל / מושהה / הושלם |
| תבנית | לא | האם זו תבנית ליצירה מהירה |

**סטטוסים:**
| סטטוס | צבע | תיאור |
|--------|------|--------|
| ACTIVE (פעיל) | ירוק | פרויקט פעיל |
| ON_HOLD (מושהה) | כתום | פרויקט מוקפא |
| COMPLETED (הושלם) | כחול | פרויקט שהסתיים |

**אוטומציה:**
- כאשר כל המשימות בפרויקט מסומנות כ"הושלם", הפרויקט מתעדכן אוטומטית ל-COMPLETED
- כאשר משימה חוזרת למצב פעיל, הפרויקט חוזר ל-ACTIVE

**תגובות על פרויקט:**
- לכל פרויקט טאב תגובות בדיאלוג הפרטים
- כל משתמש מאומת יכול להוסיף תגובה

**מחיקת פרויקט:**
- מחיקה מדורגת (Cascading Delete) בטרנזקציה: הקצאות משימות → רשומות זמן → תגובות → קבצים → היסטוריית שינויים → משימות → פרויקט
- עובד (EMPLOYEE) יכול למחוק רק פרויקט ריק (ללא משימות)
- מנהל (ADMIN) יכול למחוק כל פרויקט

| עדיפות | אוטומטי | ברירת מחדל: רגילה |

**עדיפויות:**
| עדיפות | צבע | תיאור |
|---------|------|--------|
| URGENT_IMPORTANT (דחוף וחשוב) | אדום (#f44336) | רמה 1 — עדיפות עליונה |

**סטטוסים:**
| סטטוס | צבע | תיאור |
|--------|------|--------|
| NEW (חדש) | אפור (#9e9e9e) | משימה חדשה |
| IN_PROGRESS (בביצוע) | כחול (#2196f3) | עבודה פעילה |
| WAITING_CLIENT (ממתין ללקוח) | כתום (#ff9800) | ממתין לתגובת הלקוח |
| COMPLETED (הושלם) | ירוק (#4caf50) | הסתיימה |

**5 תצוגות:**

1. **טבלה** — תצוגת ברירת מחדל. עמודות: כותרת, פרויקט, סטטוס, עדיפות, מוקצה ל, Deadline, פעולות. תומכת בסינון לפי פרויקט, סטטוס ועדיפות.

2. **קנבן (Kanban)** — 4 עמודות לפי סטטוס. גרירת כרטיס בין עמודות משנה סטטוס אוטומטית דרך ה-API. כל כרטיס מציג: כותרת, פרויקט, עדיפות, Deadline, אווטרי מוקצים.

3. **לוח שנה (Calendar)** — תצוגה חודשית. משימות מוצגות לפי תאריך ה-Deadline. עד 3 משימות לתא, צבע לפי עדיפות. ניווט בין חודשים + כפתור "היום".

4. **גאנט (Gantt)** — ציר זמן אופקי. כל שורה = משימה. רצועה מתאריך היצירה עד ה-Deadline. צבע לפי סטטוס. קו אנכי אדום מסמן את "היום". כותרות שבועיות.

5. **צוות (Team View)** — כרטיס לכל עובד. מציג: שם, תפקיד, מספר משימות פעילות, יחס הושלמו/סה"כ, פס התקדמות (ירוק ≥80%, צהוב ≥50%, כחול <50%), עד 4 משימות פעילות עם סטטוס.

**שכפול משימה:**
- יוצר עותק עם "(עותק)" בכותרת
- מעתיק את אותם מוקצים
- סטטוס מאופס ל-NEW

**היסטוריית שינויים:**
- כל פעולה (יצירה, עדכון, מחיקה) נרשמת ב-TaskChangeHistory
- שדות מעוקבים: כותרת, תיאור, סטטוס, עדיפות, Deadline, הערכת זמן
- בעדכון — נשמר ערך ישן וחדש לכל שדה שהשתנה

**אוטומציה (Subtasks):**
- כאשר כל משימות-הבת מסומנות כ"הושלם" — משימת האב מתעדכנת אוטומטית ל-COMPLETED
- כאשר משימת-בת חוזרת למצב פעיל — משימת האב חוזרת ל-IN_PROGRESS
- האוטומציה מדורגת כלפי מעלה (רקורסיבית) עד לרמת הפרויקט

**מחיקת משימה:**
- מחיקה רקורסיבית — כל משימות-הבת ותתי-הבת
- מוחק: הקצאות, רשומות זמן, תגובות, קבצים, היסטוריית שינויים, התראות

---

### 5.5 מעקב זמן

**טיימר חי:**
- הפעלה על משימה ספציפית (המשתמש חייב להיות מוקצה)
- מגבלה: עד 3 טיימרים פעילים בו-זמנית לכל משתמש
- עצירה מחשבת משך בדקות (מינימום דקה אחת)
- אירועי Socket.IO — כל המוקצים למשימה מקבלים עדכון בזמן אמת

**הזנה ידנית:**
- שדות: משימה, תאריך, שעת התחלה, שעת סיום, תיאור
- מגבלות: מקסימום 12 שעות, בדיקת חפיפה עם רשומות קיימות
- ADMIN ו-EMPLOYEE בלבד

**צפייה:**
- ADMIN רואה את כל הרשומות (יכול לסנן לפי עובד)
- EMPLOYEE רואה רק את הרשומות שלו
- CLIENT רואה רשומות של משימות המוקצות לו (צפייה בלבד)

---

### 5.6 תגובות וקבצים

**תגובות:**
- ניתן להוסיף תגובות על משימות ועל פרויקטים
- כל משתמש מאומת יכול להגיב
- התגובה כוללת: תוכן, מחבר (שם + תפקיד), תאריך
- אופציונלי: צירוף קובץ לתגובה

**קבצים:**
- העלאה דרך ממשק ייעודי (multipart/form-data)
- מגבלת גודל: 50MB לקובץ
- סוגי קבצים מותרים: jpg, jpeg, png, gif, pdf, doc, docx, xls, xlsx, txt
- אימות כפול: MIME type + סיומת קובץ
- הקבצים לא נגישים ישירות — הורדה רק דרך endpoint מאומת עם הגנת Path Traversal
- CLIENT יכול להוריד קבצים רק של משימות המוקצות לו

### 5.7 התראות

**9 סוגי התראות:**
| סוג | תיאור | מקור |
|------|--------|------|
| NEW_TASK | משימה חדשה | יצירת משימה |
| TASK_ASSIGNMENT | הקצאת משימה | הקצאה חדשה |
| STATUS_CHANGE | שינוי סטטוס | עדכון משימה |
| PRIORITY_CHANGE | שינוי עדיפות | עדכון משימה |
| NEW_COMMENT | תגובה חדשה | הוספת תגובה |
| NEW_FILE | קובץ חדש | העלאת קובץ |
| DEADLINE_APPROACHING | Deadline מתקרב | Scheduler (תוך 24 שעות) |
| DEADLINE_EXCEEDED | חריגת Deadline | Scheduler (עבר ה-Deadline) |
| ACTIVE_TIMER | טיימר שכוח | Scheduler (כל 15 דקות) |

**פאנל התראות:**
- פעמון בסרגל העליון עם Badge (מספר התראות שלא נקראו)
- לחיצה פותחת תפריט עם 50 ההתראות האחרונות
- כפתור "סמן הכל כנקראו"
- לחיצה על התראה מסמנת אותה כנקראה

**Scheduler (משימות מתוזמנות):**
- בדיקת טיימרים שכוחים — כל 15 דקות. סף ברירת מחדל: 120 דקות (ניתן לשינוי בהגדרות מערכת)
- בדיקת Deadline מתקרב — כל שעה. חלון ברירת מחדל: 24 שעות (ניתן לשינוי)
- בדיקת חריגת Deadline — כל שעה
- מניעת כפילויות: התראה אחת לטיימר לשעה, התראה אחת ל-Deadline ליום
- אופציונלי: שליחת מייל בנוסף להתראה בממשק (כאשר SMTP מוגדר)

### 5.8 דוחות

**4 סוגי דוחות:**

1. **דוח שעות** — סינון לפי טווח תאריכים, עובד, פרויקט. מציג: רשימת רשומות זמן, סיכום שעות כולל, התפלגות לפי פרויקט (גרף עמודות אופקי), התפלגות לפי עובד.

2. **סטטוס פרויקטים** — כרטיס לכל פרויקט: שם, לקוח, סטטוס, אחוז השלמה (פס התקדמות), פירוט משימות (סה"כ / הושלמו / בביצוע / חדשות / ממתינות), שעות מוערכות מול בפועל.

3. **ביצועי עובדים** — ADMIN/EMPLOYEE בלבד. סינון לפי טווח תאריכים. טבלה: שם עובד, סה"כ שעות, סה"כ משימות, הושלמו, אחוז השלמה, אחוז בזמן. גרף עמודות אנכי (שעות לפי עובד).

4. **משימות פתוחות** — כרטיסי סיכום: סה"כ פתוחות, חדשות, בביצוע, ממתינות ללקוח, באיחור. טבלה: כותרת, פרויקט, לקוח, סטטוס, עדיפות, Deadline (אדום אם עבר), מוקצה ל.

**ייצוא:**
- **PDF** — כותרת + טבלה עם רקע כחול לכותרת עמודות, יישור ימין (RTL)
- **Excel** — קובץ .xlsx עם שם גיליון בעברית, רוחב עמודות אחיד
- **CSV** — עם BOM לתצוגה תקינה של עברית ב-Excel, Escaping לגרשיים ופסיקים

## 9. שליחת מיילים (SMTP)

**תנאי:** רק כאשר `SMTP_ENABLED=true` בהגדרות הסביבה.

3 סוגי מיילים:
1. **איפוס סיסמה** — קישור חד-פעמי, תוקף שעה
2. **טיימר שכוח** — שם המשימה ומשך הזמן שעבר
3. **Deadline** — Deadline מתקרב (24 שעות) או חריגת Deadline

כל המיילים בעברית. כישלון שליחה לא קורס את האפליקציה (error swallowing).
